Bandsheet workflow summary — HTML files in root folder and commands reference

📁 HTML Files — Root Folder
🚫 Never edit directly
_template.html
Bandsheet/_template.html
Master template v2.1 — Claude reads this, injects data, and writes to a new song file. Never write songs into this file.
🚫 Never edit manually
index.html
Bandsheet/index.html
Main landing page — shows 5 band cards + global search. Reads song counts from localStorage (auto-populated when user visits band pages). No SONGS array — not updated by update_index.py.
📁 HTML Files — Band Folders
⚙️ Auto-updated
{band}/index.html
e.g. ti-muse/index.html
Song list for each band. Contains var SONGS = [...]; // -- END SONGS -- — updated automatically by update_index.py inside push.sh. Never edit SONGS by hand.
🤖 Claude creates
{band}/{song}.html
e.g. ti-muse/sleepwalk.html
Individual band sheet. Created by Claude: read _template.html → inject metadata + chords → write new file. After writing, run push.sh.
📤 Push trigger
push.sh
Bandsheet/push.sh
Run after ANY file change. Calls update_index.py → git add → git commit → git push. One command does everything.

Manual run (Terminal)
cd ~/Desktop/Vaults/Vault_ti.muse/Bandsheet && bash push.sh
update_index.py — never run this manually. It runs inside push.sh automatically. It scans all band folders, extracts metadata from each song HTML, and rewrites the var SONGS array in each {band}/index.html.
